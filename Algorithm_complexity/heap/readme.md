linked_list_sorted.py

~~~sql
Куча (heap) - это структура данных, которая представляет собой бинарное дерево с определенными свойствами. В этом случае мы реализуем мин-кучу, где значение каждого узла не меньше, чем значения его дочерних узлов. Куча обычно реализуется с использованием массива или списка, но здесь мы реализуем кучу на основе отсортированного связного списка.

Сложность операций кучи на основе отсортированного связного списка:
Вставка элемента (метод insert): Сложность O(n), где n - количество элементов в куче. Вставка происходит в отсортированный связный список, что может потребовать прохода по всему списку.
Извлечение минимального значения (метод extract_min): Сложность O(1). Минимальное значение всегда находится в корне кучи, поэтому его извлечение происходит за постоянное время.
~~~

linked_list_no_sorted.py

~~~sql
Куча (heap) - это структура данных, которая представляет собой бинарное дерево с определенными свойствами. В данном случае, мы реализуем мин-кучу, где значение каждого узла не меньше, чем значения его дочерних узлов. Здесь куча реализуется на основе неотсортированного связного списка.

Извлечение минимального значения (метод extract_min): Сложность O(n), где n - количество элементов в куче. Необходимо просмотреть все элементы списка для поиска минимального значения.
~~~

binary_heap.py

~~~sql
Бинарная куча (binary heap) - это двоичное дерево, у которого выполнены следующие свойства:

Структура кучи - полное двоичное дерево, то есть все уровни заполнены, кроме, возможно, последнего уровня, и узлы заполняются слева направо.
Значение каждого узла не меньше (для max-кучи) или не больше (для min-кучи) значения его дочерних узлов.
Бинарные кучи обычно реализуются в виде массива, где узлы хранятся в порядке уровней, начиная с корня. В массиве индексация узлов происходит следующим образом:

Индекс левого дочернего узла узла с индексом i: 2*i + 1
Индекс правого дочернего узла узла с индексом i: 2*i + 2
Индекс родительского узла для узла с индексом i: (i-1) // 2

Вставка нового значения (метод insert): Сложность O(log n), где n - количество элементов в куче. В худшем случае нужно поднять новое значение до корня, что занимает O(log n) времени.
Извлечение минимального значения (метод extract_min): Сложность O(log n), где n - количество элементов в куче. В худшем случае нужно просеять значение из корня до листа, что занимает O(log n) времени.
Проверка кучи на пустоту (метод is_empty): Сложность O(1). Это простая проверка наличия элементов в массиве.
~~~

binomial_heap.py

~~~sql
Биномиальная куча (binomial heap) - это структура данных, представляющая собой набор биномиальных деревьев. Биномиальное дерево порядка k определяется рекурсивно: это дерево, состоящее из одного узла, имеющего k потомков, каждый из которых также является биномиальным деревом порядка k-1, k-2, ..., 0 (пустое дерево). Биномиальные деревья обладают особенностью, что количество узлов в дереве порядка k равно 2^k.

Биномиальная куча представляет собой множество биномиальных деревьев, упорядоченных по возрастанию порядка, и каждое дерево должно удовлетворять свойству мин-кучи (значение узла не больше значения его потомков).

Операции над биномиальной кучей:

Вставка нового элемента.
Извлечение минимального элемента.
Объединение двух биномиальных куч.

Сложность операций биномиальной кучи:

Вставка нового элемента (метод insert): Сложность O(log n), где n - количество элементов в куче. В худшем случае нужно объединить два дерева одинакового порядка до достижения корня, что занимает O(log n) времени.
Извлечение минимального элемента (метод extract_min): Сложность O(log n), где n - количество элементов в куче. В худшем случае нужно выполнить объединение деревьев, что занимает O(log n) времени.
Проверка кучи на пустоту (метод is_empty): Сложность O(1). Это простая проверка наличия элементов в куче.
~~~

Fibonacci_heap.py

~~~sql
Фибоначчиева куча (Fibonacci heap) - это структура данных, которая представляет собой совокупность мин-куч биномиальных деревьев. Каждый элемент кучи имеет указатель на своего родителя, одного из своих потомков и своего следующего и предыдущего брата в списке уровня.
Фибоначчиева куча обладает следующими особенностями:
Быстрая вставка новых элементов: O(1) амортизированное время.
Операции объединения и удаления минимального элемента также выполняются за O(1) амортизированное время.
Операции уменьшения ключа и удаления произвольного элемента выполняются за O(log n) амортизированное время, где n - количество элементов в куче.
Вставка нового элемента (метод insert): Сложность O(1) амортизированное время.
Извлечение минимального элемента (метод extract_min): Сложность O(log n) амортизированное время, где n - количество элементов в куче.
Уменьшение ключа (метод decrease_key): Сложность O(1) амортизированное время.
Удаление произвольного элемента (метод delete): Сложность O(log n) амортизированное время, где n - количество элементов в куче.
~~~
